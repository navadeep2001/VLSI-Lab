🔢 Number Specifications

In Verilog, numbers can be written in various bases (binary, octal, decimal, hexadecimal), and can be sized or un-sized.

📏 Sized Numbers:

These specify the number of bits, base, and the value.

🧾 Syntax:

<size>’<radix><number>
  • <size> = number of bits
  • <radix> = b (binary), o (octal), d (decimal), h (hex)

✅ Examples:

  • 4’b1111 → 4-bit binary number
  • 12’habc → 12-bit hexadecimal = 12’b1010_1011_1100
  • 12’h2x5 → 12’b0010_xxxx_0101 (contains unknown x)
  • 12’o1zx → 12’b000_001_zzz_xxx (octal number with unknowns)

📏 Number Specification: Un-Sized Numbers

When size is not specified, tools generally default it to 32-bits.

🧾 Syntax:

’<radix><number>

✅ Examples:

  • ’b1111 → 32-bit binary
  • ’o21 → 32-bit octal
  • ’d225 → 32-bit decimal
  • ’h13x → 32-bit hexadecimal with unknown x

🔎 Note:

  • Default size = 32 bits, unless explicitly specified.

➖ Negative Numbers:

  • -6’d3 → 6-bit representation of -3
  • 4’d-2 → 4-bit representation of -2

🔤 Underscore Character:

  • Allowed only for readability
  • Ignored during simulation
  • Example: 12’b1111_0000_1010 is same as 12’b111100001010

❓ Question Mark:

  • '?' is treated the same as 'z' in numbers
  • Example: 4’b10?? = 4’b10zz

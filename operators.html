üß† Operators in Verilog

Operators in Verilog are used to perform arithmetic, logical, relational, and bit-level operations. They are crucial for modeling digital systems both behaviorally and structurally.

‚úÖ Types of Operators in Verilog

Verilog operators are classified into three categories:

  • Unary operators: Act on one operand
  • Binary operators: Act on two operands
  • Ternary operator: Acts on three operands (conditional)

üîπ 1. Unary Operators (Single Operand)

!    Logical NOT       // !a ‚Üí Inverts logical value
~    Bitwise NOT       // ~a ‚Üí Bitwise inversion
&    Reduction AND     // &a ‚Üí ANDs all bits
|    Reduction OR      // |a ‚Üí ORs all bits
^    Reduction XOR     // ^a ‚Üí XORs all bits
~&   Reduction NAND    // ~&a ‚Üí NAND reduction
~|   Reduction NOR     // ~|a ‚Üí NOR reduction
~^ or ^~ Reduction XNOR // ~^a ‚Üí XNOR reduction

üîç Example:

reg [3:0] a = 4'b1010;
initial begin
  $display("!a = %b", !a);   // 0
  $display("~a = %b", ~a);   // 0101
  $display("&a = %b", &a);   // 0
end

üîπ 2. Binary Operators (Two Operands)

‚ûï Arithmetic Operators

+    Addition        a + b
-    Subtraction     a - b
*    Multiplication  a * b
/    Division        a / b
%    Modulus         a % b

reg [7:0] a = 8, b = 3;
initial begin
  $display("a / b = %0d", a / b);  // 2
  $display("a %% b = %0d", a % b); // 2
end

üßÆ Bitwise Operators

&    Bitwise AND    a & b
|    Bitwise OR     a | b
^    Bitwise XOR    a ^ b
~^ or ^~ Bitwise XNOR a ~^ b
~    Bitwise NOT    ~a

reg [3:0] a = 4'b1100, b = 4'b1010;
initial begin
  $display("a & b = %b", a & b);  // 1000
  $display("a | b = %b", a | b);  // 1110
  $display("a ^ b = %b", a ^ b);  // 0110
end

üîò Logical Operators

!    Logical NOT    !a
&&   Logical AND    a && b
||   Logical OR     a || b

a = 1; b = 0;
$display("a && b = %b", a && b);  // 0
$display("a || b = %b", a || b);  // 1

üßæ Relational Operators

<    Less than         a < b
>    Greater than      a > b
<=   Less or equal     a <= b
>=   Greater or equal  a >= b

üü∞ Equality Operators

==   Equal (ignores x/z)   a == b
!=   Not equal             a != b
===  Case equality         a === b
!==  Case inequality       a !== b

reg [1:0] a = 2'b1x, b = 2'b1z;
$display("a == b  = %b", a == b);     // X
$display("a === b = %b", a === b);    // 0

ü™Ñ Shift Operators

<<   Logical Left Shift      a << 2
>>   Logical Right Shift     a >> 2
<<<  Arithmetic Left Shift   a <<< 2
>>>  Arithmetic Right Shift  a >>> 2

a = 4'b1100;
$display("a >> 1 = %b", a >> 1);  // 0110

üîπ 3. Ternary Operator

assign y = (sel) ? a : b;

If sel is true (1), assign a to y; else assign b.

üîÑ Other Operator Categories

üß© Concatenation

a = 4'b1100; b = 4'b0011;
y = {a, b};   // 8'b11000011
x = {4{1'b1}}; // 4-bit 1111

üîÇ Reduction Operators

a = 4'b1110;
$display("&a = %b", &a);  // 0
$display("|a = %b", |a);  // 1
$display("^a = %b", ^a);  // 1 (1^1^1^0)

üß† Operator Precedence

1 (High): () {}
2: ~, !, unary +, -, reduction
3: *, /, %
4: +, -
5: <<, >>, <<<, >>>
6: <, <=, >, >=
7: ==, !=, ===, !==
8: & (bitwise AND)
9: ^, ^~, ~^
10:
11: &&
12:
13: ? : (ternary)
14 (Low): =, <=

Tip: Always use parentheses () for clarity in complex expressions.

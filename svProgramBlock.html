üî∑ What is a program block?

A program block in SystemVerilog is specifically designed for testbenches. It provides a race-free interface between the testbench and the design (DUT). This block is executed in a special Reactive region of the simulation time wheel, after the design‚Äôs logic (which is executed in the Active region), thus avoiding race conditions.

üß† Why use program block over module?

In SystemVerilog, the design code (DUT) resides in module blocks and is executed in the Active region.

If you write your testbench in a module, both the DUT and testbench might evaluate simultaneously, leading to race conditions.

But when the testbench is inside a program block, its execution is scheduled after the design's, which ensures:

  • All signal changes in the DUT are visible to the testbench.
  • No simultaneous execution ‚áí Race-free behavior.

üîß program block Syntax

program test(input clk, input [7:0] addr, output [7:0] wdata);
    ...
endprogram

// OR using interface:
program test (interface mem_intf);
    ...
endprogram

‚öîÔ∏è program block vs module block ‚Äì Example Comparison

Example 1: Using a regular module for testbench

üîπ Design Code:

module design_ex(output bit [7:0] addr);
  initial begin
    addr <= 10;
  end   
endmodule

üîπ Testbench (as module):

module testbench(input bit [7:0] addr);
  initial begin
    $display("\t Addr = %0d", addr);
  end
endmodule

üîπ Top-level:

module tbench_top;
  wire [7:0] addr;

  design_ex dut(addr);
  testbench test(addr);
endmodule

üîπ Simulator Output:

Addr = 0

üî¥ Why? Because testbench and DUT both execute in the Active region, there's a race condition, and testbench captures the value before it is updated.

Example 2: Using a program block for testbench

üîπ Testbench (as program):

program testbench(input bit [7:0] addr);
  initial begin
    $display("\t Addr = %0d", addr);
  end
endprogram

üîπ Top-level (unchanged):

module tbench_top;
  wire [7:0] addr;

  design_ex dut(addr);
  testbench test(addr);
endmodule

üîπ Simulator Output:

Addr = 10

‚úÖ Why? program block executes in the Reactive region, which ensures the DUT updates the value first, and then the testbench reads it ‚áí race-free interaction.

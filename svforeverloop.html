A forever loop runs infinitely ‚Äî it will never stop unless there‚Äôs a break or end condition provided inside. It‚Äôs typically used in verification environments where continuous monitoring or repeated behavior is needed.

üîπ Example 1: Basic forever loop with a delay

  end

üß† Why use forever instead of always?

In SystemVerilog classes, you can't use always blocks. So when modeling monitors, drivers, or other behavioral components, forever inside a task is the go-to way.

üîπ Example 2: Forever loop in a class-based monitor

class BusMonitor;
  virtual interface vif;

  function new(virtual interface vif);
    this.vif = vif;
  endfunction

  task run();
    forever begin
      @(posedge vif.clk); // Wait for clock edge
      if (vif.en && vif.valid) begin
        $display("Transaction seen: data = %0h", vif.data);
      end
    end
  endtask
endclass

module tb;
  interface bus_if(input logic clk);
    logic en;
    logic valid;
    logic [7:0] data;
  endinterface

  logic clk = 0;
  bus_if vif(clk);
  BusMonitor monitor;

  always #5 clk = ~clk; // Clock generation

  initial begin
    monitor = new(vif);
    fork
      monitor.run(); // Start monitor task
    join_none

    // Simulate some activity
    #10 vif.en = 1; vif.valid = 1; vif.data = 8'hAB;
    #10 vif.valid = 0;
    #20 $finish;
  end
endmodule

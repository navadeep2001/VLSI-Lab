üÜî Identifiers

Identifiers are names given to variables, modules, signals, etc., to reference them in code.

They make the design readable and manageable.

‚úÖ Rules for Identifiers

  • Can contain letters, digits, underscore (_), and dollar sign ($).
  • Case-sensitive (Data and data are different).
  • Maximum length: 1024 characters.
  • Must start with a letter or underscore.
  • Cannot start with a digit or just a $.

üìå Examples

reg value;     // 'value' is an identifier
input clk;     // 'clk' is an identifier
reg _data1;    // Valid
reg $temp;     // Valid (starts with letter + $ used later)
reg 1value;    // ‚ùå Invalid (starts with digit)

üîë Keywords

Keywords are reserved words in Verilog.

They define language constructs like modules, data types, control flow, etc.

Do not use keywords as identifiers unless you escape them using \.

Verilog keywords are lowercase and case-sensitive.

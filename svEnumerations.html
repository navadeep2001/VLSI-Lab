SystemVerilog Enumerations

What is an Enumeration (enum)?

An enumeration (or enum) is a user-defined data type that lets you define named values for variables instead of using raw numbers. It makes the code more readable and maintainable.

Example Without Enum (Hard to Read):

bit [1:0] light;
light = 2'b00;  // What does 00 mean? RED, GREEN, or YELLOW?

if (light == 2'b00) begin
    // Do something
end

üî¥ Problem: After a few lines of code, it's hard to remember what 00, 01, or 10 represent.

Example With Enum (Easy to Read):

typedef enum {RED, YELLOW, GREEN} e_light;
e_light light;

light = RED;  // Directly assign RED
if (light == RED) begin
    // Do something
end

‚úÖ Better Readability: We can clearly see that light is set to RED, making the code more intuitive.

Defining an Enum in SystemVerilog

1Ô∏è‚É£ Direct Declaration (Without Typedef):

enum {RED, YELLOW, GREEN} light;

Here, light is declared as an enum that can take values RED (0), YELLOW (1), or GREEN (2).

However, this method is not reusable because we can't use the same enum type for another variable.

2Ô∏è‚É£ Using typedef (Recommended for Reusability):

typedef enum {RED, YELLOW, GREEN} e_light;
e_light light1, light2;

e_light is a new data type, and we can declare multiple variables (light1, light2) of this type. This is reusable and good coding practice.

Assigning Custom Values

enum {RED = 3, YELLOW, GREEN} light;
enum {RED = 4, YELLOW = 9, GREEN} light2;

üî¥ Mistake to Avoid: Duplicate Values

enum {RED = 2, YELLOW, GREEN = 3} light;  // ‚ùå ERROR: YELLOW & GREEN both have 3!

Illegal Enumeration Mistakes & How to Fix Them

üö® Mistake 1: Incorrect Enum Declaration

module test();
    int a; {RED, GREEN} colour;  // ‚ùå ERROR
    int b;
    enum (RED, GREEN) colourO;  // ‚ùå ERROR
endmodule

‚úîÔ∏è Corrected Code:

module test();
    typedef enum {RED, GREEN} eColour;  
    eColour colourO;  
endmodule

üö® Mistake 2: Redefining Enum Values

module test();
    enum {RED, GREEN} colour;
    initial begin
        colour = RED;
        colour = 0;  // ‚ùå ERROR
    end
endmodule

‚úîÔ∏è Corrected Code Using $cast():

module test();
    enum {RED, GREEN} colour;
    initial begin
        colour = RED;
        $cast(colour, 0);  // ‚úÖ Correct
    end
endmodule

üìå Why Use $cast()?
SystemVerilog enums only accept their predefined values. $cast() allows controlled assignment.

üö® Mistake 3: Enum Scope Issues

module test();
    enum {RED, GREEN} colour;
    enum {DAY, NIGHT} mode;
    initial begin
        colour = RED;
        colour = DAY;  // ‚ùå ERROR
    end
endmodule

‚úîÔ∏è Fix: Use Separate Variables

module test();
    typedef enum {RED, GREEN} e_colour;
    typedef enum {DAY, NIGHT} e_mode;

    e_colour colour;
    e_mode mode;

    initial begin
        colour = RED;
        mode = DAY;  // ‚úÖ Correct
    end
endmodule

Enumerated Type Range

  • name ‚Äì The next number will be associated with name
  • name = C ‚Äì Associates the constant C to name
  • name[N] ‚Äì Generates N named constants : name0, name1, ..., nameN-1
  • name[N] = C ‚Äì First named constant gets value C, rest follow
  • name[N:M] ‚Äì Range naming (e.g. name3 to name5)
  • name[N:M] = C ‚Äì Same as above, starting at C

1Ô∏è‚É£ Basic Enum (Auto-Numbering)

typedef enum {GREEN, YELLOW, RED, BLUE} color_set_1;

‚úÖ Result:

  • GREEN = 0
  • YELLOW = 1
  • RED = 2
  • BLUE = 3

2Ô∏è‚É£ Assigning Custom Values

typedef enum {MAGENTA=2, VIOLET=7, PURPLE, PINK} color_set_2;

‚úÖ Result:

  • MAGENTA = 2
  • VIOLET = 7
  • PURPLE = 8
  • PINK = 9

3Ô∏è‚É£ Numbering Multiple Colors in Bulk

typedef enum {BLACK[4]} color_set_3;

‚úÖ Result: BLACK0 to BLACK3 = 0 to 3

4Ô∏è‚É£ Bulk Naming with Custom Start Value

typedef enum {RED[3] = 5} color_set_4;

‚úÖ Result: RED0 = 5, RED1 = 6, RED2 = 7

5Ô∏è‚É£ Define a Range with Custom Naming

typedef enum {YELLOW[3:5]} color_set_5;

‚úÖ Result: YELLOW3 = 0, YELLOW4 = 1, YELLOW5 = 2

6Ô∏è‚É£ Range with Custom Start Value

typedef enum {WHITE[3:5] = 4} color_set_6;

‚úÖ Result: WHITE3 = 4, WHITE4 = 5, WHITE5 = 6

Enumerated Type Methods

  • first() ‚Äì Returns the first enum value
  • last() ‚Äì Returns the last enum value
  • next() ‚Äì Returns the next enum value
  • prev() ‚Äì Returns the previous enum value
  • num() ‚Äì Total elements
  • name() ‚Äì String representation
typedef enum {RED, YELLOW, GREEN} traffic_light;

module tb;
  initial begin
    traffic_light light;
    light = YELLOW;

    $display ("First value: %s (%0d)", light.first().name(), light.first());
    $display ("Last value : %s (%0d)", light.last().name(), light.last());
    $display ("Next value : %s (%0d)", light.next().name(), light.next());
    $display ("Previous value : %s (%0d)", light.prev().name(), light.prev());
    $display ("Total elements: %0d", light.num());
    $display ("Current name: %s", light.name());
  end
endmodule

Expected Output

First value: RED (0)
Last value : GREEN (2)
Next value : GREEN (2)
Previous value : RED (0)
Total elements: 3
Current name: YELLOW
